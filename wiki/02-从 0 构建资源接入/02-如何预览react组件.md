# 02 | 如何拉起 Storybook

> 这一篇写给不太熟悉前端工程的设计师：如何打开 Storybook，预览组件、Block 和页面效果。

---

## 1. 先理解 Storybook 是什么

### 1.1 一句话理解

Storybook 是设计系统的“组件展厅”。

它可以把项目里的 Component、Block、页面模板单独渲染出来，让设计师不用进入真实业务系统，也能检查：

- 组件默认态、选中态、禁用态、加载态是否正确
- 颜色、间距、字号、圆角是否符合设计稿
- Block 里的元素组合和显隐是否合理
- 新接入的资源是否真的能被页面调用

### 1.2 设计师为什么需要会打开它

在资源接入流程里，Agent 会尽量自动生成代码、补 stories、修复错误。设计师最重要的动作不是写代码，而是**打开 Storybook 看真实渲染结果**。

```text
Agent 负责生成和修复
设计师负责看结果是否符合设计意图
```

如果你只看 Figma 或 benchmark HTML，还不能确认组件已经进入真实系统。只有 Storybook 能证明：这个组件已经变成可运行的 React 真值。

---

## 2. 最简单的启动方式

### 2.1 先打开终端

你可以使用任意一种终端：

- macOS 自带的“终端”
- VS Code / Cursor 底部的 Terminal
- 让 Agent 帮你执行命令

如果你不熟悉终端，可以直接对 Agent 说：

```text
请帮我在 harmony-ui-playground 目录下拉起 Storybook，并把本地预览地址发给我。
```

### 2.2 进入项目目录

如果你自己操作，先进入项目里的预览工程：

```bash
cd harmony-ui-playground
```

你可以把它理解成：先走进放组件展厅的文件夹。

### 2.3 安装依赖

第一次打开项目时，需要先安装依赖。

```bash
pnpm install
```

如果你的电脑没有 `pnpm`，也可以让 Agent 帮你处理。不要在不确定时反复乱装，直接把报错发给 Agent 即可。

### 2.4 启动 Storybook

```bash
pnpm storybook
```

启动成功后，终端里通常会出现一个本地地址：

```text
http://localhost:6006
```

把这个地址复制到浏览器里打开，就能看到 Storybook。

---

## 3. 打开后怎么看

### 3.1 左侧导航

Storybook 左侧是目录导航。常见内容包括：

| 区域 | 用来看什么 |
|---|---|
| `Components/*` | 单个组件，例如 Button、Switch、StatusBar |
| `Blocks/*` | 由多个组件组成的稳定区块 |
| `Foundations/*` | 颜色、字体、间距、圆角、阴影等基础样式 |
| `Showcase` | 多个组件的总览页面，用于快速 smoke check |

不同项目的目录名可能略有差异，但阅读方式一样：**左边选对象，右边看真实渲染结果。**

### 3.2 右侧预览区

右侧是组件实际渲染出来的样子。设计师主要看：

- 是否和设计稿一致
- 是否缺少状态或变体
- 文案、图标、间距、圆角是否跑偏
- 切换 props 后样式是否仍然正确

### 3.3 Controls 面板

Storybook 通常会有 Controls 面板，可以切换组件 Props。

例如一个 Button 可能可以切换：

| 控制项 | 用途 |
|---|---|
| `variant` | 看 primary / secondary / danger 等变体 |
| `size` | 看不同尺寸 |
| `disabled` | 看禁用态 |
| `loading` | 看加载态 |

如果 Controls 里没有你期望的状态，通常说明 story 没补全，或者组件 Props 还没有暴露出来。

## 4. 验收时看什么

### 4.1 Component 验收

检查单个 Component 时，重点看：

| 验收项 | 要看什么 |
|---|---|
| 视觉 | 颜色、字号、间距、圆角、阴影是否符合设计稿 |
| 状态 | 默认、hover、active、disabled、loading、error 是否覆盖 |
| 变体 | variant 命名和设计语义是否一致 |
| 尺寸 | 不同 size 是否稳定，不会挤压或溢出 |
| 交互 | 开关、输入、选择、展开收起是否符合预期 |

### 4.2 Block / Page 验收

检查 Block 或 Page 时，重点看：

| 验收项 | 要看什么 |
|---|---|
| 结构 | 区块顺序、层级、信息组织是否符合设计 |
| 显隐 | 可选区域隐藏后是否仍然自然 |
| 替换 | slot 替换 Block 后页面是否仍然成立 |
| 响应式 | 移动端 / 桌面端预览是否没有明显错位 |
| 复用性 | 这个 Block 是否像稳定模板，而不是一次性页面截图 |

## 5. 常见问题

### 5.1 端口 6006 被占用

如果终端提示 6006 被占用，可以让 Agent 换一个端口，例如：

```text
Storybook 端口被占用了，请帮我换一个可用端口拉起，并告诉我新的预览地址。
```

### 5.2 页面打不开

先确认三件事：

| 检查项 | 怎么判断 |
|---|---|
| 终端是否还在运行 | 终端里还停留在 Storybook 输出，不要关闭窗口 |
| 地址是否正确 | 通常是 `http://localhost:6006` |
| 是否有报错 | 如果终端出现红色错误，把报错发给 Agent |

### 5.3 样式看起来不对

样式不对时，不要自己改代码。直接描述你看到的问题：

```text
这个组件在 Storybook 里和设计稿不一致：
1. 标题字号偏大
2. 卡片圆角不对
3. disabled 状态颜色太深
请根据 Storybook 结果修正，并重新让我验收。
```

### 5.4 找不到组件

如果左侧导航里找不到组件，可能是 story 没有创建或没有被 Storybook 识别。可以让 Agent 检查：

```text
我在 Storybook 里找不到 [组件名]。
请检查它是否有 stories 文件，是否被 Storybook 正确加载。
```

## 6. 设计师最小介入清单

| 阶段 | 设计师需要做什么 |
|---|---|
| 启动前 | 确认要看哪个组件、Block 或页面 |
| 启动时 | 可以让 Agent 拉起 Storybook，并提供本地地址 |
| 预览时 | 打开浏览器检查真实渲染效果 |
| 验收时 | 说明 PASS / FAIL；FAIL 时描述具体不一致点 |

除此之外，安装依赖、启动服务、修复构建错误、补 story，都可以交给 Agent 自动完成。

---

## 7. 推荐给 Agent 的说法

### 7.1 只想打开预览

```text
请在 harmony-ui-playground 目录下拉起 Storybook。
如果端口 6006 被占用，请换一个可用端口。
启动成功后，把本地预览地址告诉我。
```

### 7.2 想验收某个组件

```text
请拉起 Storybook，并帮我定位到 [组件名] 的 story。
我要检查它的默认态、禁用态、加载态和所有 variant。
```

### 7.3 验收失败后修正

```text
Storybook 里 [组件名] 没有通过验收。
问题是：[具体问题]。
请修正组件或 story，重新拉起 Storybook 后让我再次检查。
```
