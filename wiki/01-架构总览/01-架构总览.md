## 四层架构

| 层级 | 目录 | 职责 | 可替换 |
|------|------|------|--------|
| Agent Layer | `.agent/` | Skill 规则和工作流定义 | 否 |
| Resource Layer | `.resources/` | 设计系统索引和配置 | **是** |
| Source Layer | `{project}/` | 真实代码来源（组件、Block、样式） | **是，由你维护** |
| Wiki | `wiki/` | 项目文档 | 否 |

## 页面生成工作流

当用户输入 prompt 请求生成页面时，Agent 按以下步骤执行：

输入 prompt
    ↓
┌─────────────────────────────────────┐
│ [1] 读取项目上下文                  │
│     - 读取 components.json           │
│     - 确认别名、Tailwind、icon 库   │
└─────────────────────────────────────┘
    ↓
┌─────────────────────────────────────┐
│ [2] Route Matching                  │
│     - 读取 route-index.md           │
│     - 匹配请求 → page_type          │
└─────────────────────────────────────┘
    ↓
┌─────────────────────────────────────┐
│ [3] Layout Resolution               │
│     - 读取 layout/index.md          │
│     - 读取 layout/{page_type}.md   │ ← 必定获取
│     - 提取 layout_skeleton          │
│       reference_blocks              │
│       needed_components            │
└─────────────────────────────────────┘
    ↓
┌─────────────────────────────────────┐
│ [4] Source Grounding                │
│     - 读取 blocks.json              │
│     - 定位 blocks/*.tsx 源码         │
│     - 读取 components.json          │
│     - 定位 component/*.tsx 源码     │
│     - 读取 component/*.md           │ ← 仅在 tsx 源码不足以
│       (仅在需要时)                  │   解释组件行为时获取
└─────────────────────────────────────┘
    ↓
┌─────────────────────────────────────┐
│ [5] 生成页面                        │
│     - 读取 tsx 源码                 │
│     - 注入 template                 │
│     - 应用 tokens                   │
└─────────────────────────────────────┘
    ↓
┌─────────────────────────────────────┐
│ [6] 验证与日志                      │
│     - 运行 validate 脚本            │
│     - 写入 generation log           │
└─────────────────────────────────────┘

## 文件调用关系

当 AI 生成一个页面时，文件的读取顺序如下：

```
1. 读取 .resources/config.json
   → 确定当前激活的设计系统（如 harmony）

2. 读取 .resources/{active}/route-index.md
   → 根据用户需求匹配 page_type

3. 读取 .resources/{active}/layout/{page_type}.md
   → 获取该页面类型的布局骨架

4. 读取 .resources/{active}/blocks.json
   → 找到对应的 Block 源码路径

5. 读取 {project}/src/blocks/{block}.tsx
   → 使用 Block 源码生成页面（Block 源码由你维护）

6. 读取 {project}/src/component/*.tsx
   → 使用组件源码组装 Block（组件源码由你维护）
```

整个生成过程依赖你维护的两层内容：资源层的索引配置 + 源码层的实际代码。

## 你需要维护的内容

作为资源层维护者，你维护两层内容：

### 资源层（`.resources/`）

| 文件 | 作用 | 维护频率 |
|------|------|----------|
| `config.json` | 指定激活哪个设计系统 | 切换系统时 |
| `route-index.md` | 定义所有页面类型 | 新增 page_type 时 |
| `layout/*.md` | 每种 page_type 的布局模板 | 新增布局时 |
| `blocks.json` | Block 名称、路径、依赖 | 新增 Block 时 |
| `components.json` | 组件名称、路径、别名 | 新增组件时 |
| `component/*.md` | 组件的详细说明 | 可选 |

### 源码层（`{project}/src/`）

| 文件 | 作用 | 维护频率 |
|------|------|----------|
| `component/*.tsx` | 组件源码（按钮、开关、列表等） | 新增/修改组件时 |
| `blocks/*.tsx` | Block 页面模板源码 | 新增/修改 Block 时 |
| `styles/` | 设计 token 和全局样式 | 调整设计规范时 |
