# 01 | 设计师实操手册

> 这一篇是 Design System Skill 的完整操作指南，包含检查清单和话术模板。

---

## 什么时候走这个流程？

当你需要：
- 接入一个新的设计系统
- 生成一个完整页面
- 维护设计系统资源

---

## 完整路径

```
选择任务类型
    ↓
按任务类型执行对应流程
    ↓
设计师验收
    ↓
完成
```

---

## 每一步谁做什么

🤖 = AI 独立完成
👤 = 设计师独立完成
🤝 = AI 和设计师协作

---

## 提交前检查清单

- [ ] 设计系统信息已提供
- [ ] config.json 已配置
- [ ] route-index.md 已创建/更新
- [ ] layout markdown 已创建/更新
- [ ] blocks.json 和 components.json 已更新
- [ ] 构建验证通过

---

## 完整任务示例

### 示例：接入新的设计系统

### Step 1：告诉 AI

```
"我要把 [设计系统名] 接入为新资源集"
```

### Step 2：AI 生成配置，你来审核

AI 会更新 config.json 和创建目录结构

### Step 3：验证

AI 运行验证

## 设计师话术模板

| 场景 | 推荐话术 |
|------|--------|
| 接入新设计系统 | "帮我把 [设计系统名] 接入为新资源集" |
| 切换设计系统 | "切换到 [设计系统名] 资源集" |
| 生成页面 | "帮我生成一个 [页面描述]" |
| 新增 Layout | "新增 [页面类型] 布局" |
| 新增 Block | "新增 [Block 名称] Block" |
| 新增 Component | "新增 [组件名称] 组件" |

## 常见问题

**Q：我不确定该走哪个流程？**
A：走 00-快速入口，用决策树判断。

**Q：构建验证失败了怎么办？**
A：把错误信息发给 AI，让它解释。
