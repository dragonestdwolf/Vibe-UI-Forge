# 99 | 术语表

> 这里集中解释你在工作中会反复遇到的术语。

---

## 核心术语

### Resource Layer — 资源层

位于 `.resources/` 目录下的设计系统索引，包含：
- config.json — 资源配置
- route-index.md — 路由索引
- layout/ — 布局规范
- blocks.json — Block 索引
- components.json — Component 索引

**设计师理解：** 资源层是设计系统的"目录"，告诉 AI 有哪些组件和布局可用。

---

### Source Layer — 源码层

位于 `harmony-ui-playground/src/` 目录下的真实代码，包含：
- component/ — 组件实现
- blocks/ — 区块实现
- styles/ — 设计 Token

**设计师理解：** 源码层是设计的"真实来源"，AI 生成代码时以这里为准。

---

### page_type — 页面类型

设计系统识别的页面分类，如：
- mobile-settings — 设置页
- health-dashboard — 健康仪表盘
- mobile-sheet — 半模态浮层

**设计师理解：** page_type 告诉 AI 当前页面属于哪种类型，以便匹配正确的布局。

---

### layout — 布局

页面类型的布局规范，定义：
- layout_skeleton — 页面骨架
- needed_components — 需要的组件
- composition_mapping — 组件映射

**设计师理解：** layout 是页面的"模具"，决定组件如何排列。

---

### block — 区块

可复用的页面模板，如：
- settings-page — 设置页模板
- health-clover — 健康仪表盘模板

**设计师理解：** block 是完整的页面模板，可以直接使用或作为参考。

---

### component — 组件

可复用的 UI 单元，如：
- StatusBar — 状态栏
- TitleBar — 标题栏
- List — 列表
- Switch — 开关

**设计师理解：** component 是最小的可复用单元，如按钮、列表、开关等。

---

### Shell — 页面骨架

页面类型的"模具"，定义页面的整体结构。

---

### Token — 设计变量

保证界面一致性的"配方本"，如颜色、字号、间距变量。

---

### Skill Manifest — 技能清单

组件的完整交付物 = Spec + Template + 元数据。

---
