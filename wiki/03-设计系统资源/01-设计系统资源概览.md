# 01 | 设计系统资源概览

> 这一篇帮助你在开工前建立全局认知——知道系统有几层、每层产什么、用户站在哪个位置。

---

## 系统全景图

```
用户输入 / Figma Design
    ↓
┌─────────────────────────────────────┐
│ Agent Layer (.agent/skills/)          │
│ 通用 Skill，与业务无关                │
└─────────────────────────────────────┘
    ↓
┌─────────────────────────────────────┐
│ Resource Layer (.resources/)          │
│ 设计系统资源索引（可替换）             │
└─────────────────────────────────────┘
    ↓
┌─────────────────────────────────────┐
│ Source Layer (harmony-ui-playground/) │
│ 真实代码来源                          │
└─────────────────────────────────────┘
```

---

## 三层职责

| 层级 | 作用 | 可替换 |
|------|------|--------|
| Agent Layer | 提供通用工作流能力 | 否 |
| Resource Layer | 设计系统的索引和配置 | ✅ 是 |
| Source Layer | 真实代码来源 | 否 |

---

## 资源层结构

```
.resources/
├── config.json                  # 资源配置
└── {design-system}/            # 设计系统资源集
    ├── route-index.md         # 路由索引
    ├── layout/                # 布局规范
    │   ├── index.md
    │   └── {page_type}.md
    ├── blocks.json            # Block 索引
    ├── components.json        # Component 索引
    └── component/             # 组件规范（按需）
        └── {name}.md
```
