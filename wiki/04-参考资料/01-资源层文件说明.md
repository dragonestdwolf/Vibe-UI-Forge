# 01 | 资源层文件说明

> 这一篇集中解释 `.resources` 目录下每类文件的职责。它是 [Design System Skill 简介](../01-认识%20Skill/02-Design%20system%20skill简介.md) 的细节参考。

---

## 1. Resource Layer 是什么

### 1.1 一句话理解

Resource Layer 是给 AI 看的设计系统地图和 how to use 说明书。

它负责回答这些问题：

- 用户这个需求应该命中哪种 `page_type`
- 这种页面应该读取哪个 layout
- layout 里应该参考哪些 Block
- 生成页面时可以使用哪些 Component
- Block / Component 有哪些使用边界和组合约束

但 Resource Layer 不是源码本身。它指向 Source Layer，真正的 Props、状态、样式和 token 仍然以 React / CSS 源码为准。

### 1.2 和其他层的关系

```text
Skill Layer 规则
    ↓
读取 Resource Layer 的 route-index → 命中 page_type
    ↓
读取 Resource Layer 的 layout/{page_type}.md → 知道用什么 Block 和 Component
    ↓
Resource Layer 的 blocks.json / components.json → 指向 Source Layer 的真值
    ↓
AI 回到 Source Layer 读源码 → 确认 Props、样式、token
    ↓
生成页面到 render/ → 验证 → 写 Generation Log
```

记住：**Skill 是规则，Resource 是地图，Source 是真值。地图可以换，真值不能漂移。**

完整项目目录见 [07-项目目录说明](./07-项目目录说明.md)。

## 2. 资源层结构

### 2.1 标准目录

```text
.resources/
├── config.json                         # 选择当前激活的设计系统
└── {design-system}/
    ├── route-index.md                  # page_type 路由索引
    ├── layout/                         # 每种 page_type 的 layout markdown
    │   ├── index.md
    │   ├── mobile-settings.md
    │   └── ...
    ├── blocks.json                     # Block 索引，指向 Source Layer 的源码
    ├── block/                          # Block 的 how to use 说明
    │   ├── settings-page.md
    │   └── ...
    ├── components.json                 # Component 索引，指向 Source Layer 的源码
    └── component/                      # Component 的 how to use 说明
        ├── button.md
        ├── switch.md
        └── ...
```

### 2.2 文件职责

| 文件 | 作用 | 详细模板 |
|---|---|---|
| `config.json` | 选择当前激活的设计系统资源集 | 本文说明 |
| `route-index.md` | 把 prompt / Figma 意图映射到 `page_type` | [route-index 模板](./02-route-index%20模板.md) |
| `layout/{page_type}.md` | 定义页面类型的布局骨架、插槽、显隐和替换规则 | [layout markdown 模板](./03-layout%20markdown%20模板.md) |
| `blocks.json` | 注册稳定 Block 模板，并指向真实源码 | [blocks.json 模板](./04-blocks.json%20模板.md) |
| `block/*.md` | 描述 Block 的组件嵌套、状态同步、适用场景和使用边界 | 参考 Block 抽取章节 |
| `components.json` | 注册可用 React Component，并指向真实源码 | [components.json 模板](./05-components.json%20模板.md) |
| `component/*.md` | 补充复杂组件的语义、变体、状态和 how to use 说明 | 参考 Component 抽取章节 |

## 3. 核心文件怎么看

### 3.1 config.json

`config.json` 决定当前激活哪套设计系统资源。

```json
{
  "active": "harmony",
  "resources": {
    "harmony": {
      "path": ".resources/harmony",
      "projectRoot": "harmony-ui-playground"
    }
  }
}
```

### 3.2 route-index.md

`route-index.md` 把用户请求或 Figma 页面意图匹配成 `page_type`。它只负责“命中哪类页面”，不负责描述具体页面结构。

```markdown
| page_type | layout | hit_rules | exclusion_rules |
|---|---|---|---|
| mobile-settings | layout/mobile-settings.md | 设置 / 系统设置 | 健康 |
```

### 3.3 layout/{page_type}.md

layout markdown 是 Resource Layer 里最核心的 how to use 文件。它告诉 AI 某类页面应该如何组织：

- 页面结构骨架：`layout_skeleton`
- 参考 Block：`reference_blocks`
- 需要组件：`needed_components`
- 可选显隐：`visibility_rules`
- 插槽替换：`slots` / `replacement_blocks`
- 生成约束：`generation_constraints`

具体字段见 [layout markdown 模板](./03-layout%20markdown%20模板.md)。

### 3.4 blocks.json 和 block/*.md

`blocks.json` 注册稳定 Block，并指向真实源码。`block/*.md` 补充说明这个 Block 如何使用。

两者分工是：

| 文件 | 回答的问题 |
|---|---|
| `blocks.json` | 这个 Block 的稳定 ID 是什么，源码在哪里 |
| `block/*.md` | 这个 Block 适合什么场景，哪些区域可隐藏，组件如何组合 |

### 3.5 components.json 和 component/*.md

`components.json` 注册当前资源集可使用的 Component，并指向真实源码、stories 和补充说明。

`component/*.md` 只在源码和 stories 不足以表达语义时使用，例如复杂状态、业务含义、禁用组合、设计侧命名等。

## 4. 默认读取策略

| 文件 | 默认读取吗 | 说明 |
|---|---|---|
| `config.json` | 是 | 确定 active resource |
| `route-index.md` | 是 | 做 route matching |
| `layout/{page_type}.md` | 是 | 做 layout resolution |
| `blocks.json` | 是 | 找 reference blocks |
| `components.json` | 是 | 找 needed components |
| `block/*.md` | 按需 | Block 组合关系或业务语义不清楚时读取 |
| `component/*.md` | 按需 | 源码和 stories 不足以表达语义时读取 |

---

## 5. 真值边界

```text
Markdown 资源层 = 地图和说明书
React / CSS / token = 真值
src/render/** = 生成预览，不是模板源
```

| 边界 | 说明 |
|---|---|
| `.resources/**` | 给 AI 的索引和 how to use 说明 |
| `src/component/**` | Component 的真实实现 |
| `src/blocks/**` | 稳定 Block 模板源 |
| `src/render/**` | 生成结果预览，不能注册为 reference block |
| CSS / token | 样式真值，生成页面必须对齐这里 |
