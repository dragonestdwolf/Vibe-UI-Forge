# 06 | 术语表

> 这里解释本项目中反复出现的核心术语。它用于快速查阅，不替代前面的流程文档。

---

## 1. 核心分层

| 术语 | 含义 | 常见位置 |
|---|---|---|
| Skill Layer | AI 的工作规则，定义如何读取资源、生成页面和验证结果 | `.agent/skills/**` |
| Resource Layer | 给 AI 的设计系统地图和 how to use 说明 | `.resources/**` |
| Source Layer | 真实代码来源，AI 生成时必须对齐这里 | `src/component/**`、`src/blocks/**`、`src/styles/**` |
| Wiki Layer | 给人看的说明文档，不是运行时资源 | `wiki/**` |

---

## 2. 页面生成

| 术语 | 含义 |
|---|---|
| page_type | 页面类型 ID，例如 `mobile-settings`、`health-dashboard`；它决定 AI 读取哪个 layout |
| Route Matching | 把用户需求或 Figma 意图匹配到 `page_type` 的过程 |
| Layout | 某个 `page_type` 的页面结构说明，通常位于 `.resources/{system}/layout/{page_type}.md` |
| layout_skeleton | layout 里的页面骨架，只表达结构和区域，不表达逐像素视觉 |
| slots | layout 中可替换的区域，说明某个位置可以放哪些 Block |
| visibility_rules | layout 或 Block 中的显隐规则，说明哪些区域可关闭、默认是否显示 |
| reference_blocks | layout markdown 中声明的参考 Block ID，必须能在 `blocks.json` 中找到 |
| needed_components | layout markdown 中声明的必需组件 ID，必须能在 `components.json` 中找到 |

---

## 3. 资源单元

| 术语 | 含义 |
|---|---|
| Component | 可复用 UI 单元，例如 Button、List、Switch、StatusBar |
| Block | 稳定页面区块模板，由多个 Component 组合而成，通常位于 `src/blocks/**` |
| Page | 完整页面模板，对应一个 `page_type`，通常由多个 Block 组合而成 |
| Figma Intent | 设计稿表达的页面意图、组件语义、状态和布局关系；AI 不应只复刻图层 |

---

## 4. 验证与边界

| 术语 | 含义 |
|---|---|
| Source Grounding | AI 读取真实源码、stories、CSS 和 token，并以它们作为生成依据 |
| Render Preview | 生成页面的预览输出，通常位于 `src/render/**`；只用于验收，不作为模板源 |
| Generation Log | 页面生成旁边的记录文件，通常是 `index.log.md`，记录输入、命中资源、生成文件和验证结果 |
| 真值 | 最终可信依据；在本项目中通常是 React 源码、CSS、token 和可运行 stories |
| 地图 | 帮 AI 寻路和理解语义的资源说明；通常指 Resource Layer |
