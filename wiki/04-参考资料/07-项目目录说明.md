# 07 | 项目目录说明

> 这一篇集中说明项目里的关键目录。第一次阅读不需要背目录树，只要知道每一层负责什么即可。

---

## 最小心智模型

```text
Vibe-UI-Forge-main/
├── .agent/skills/              # Skill Layer：AI 的工作规则
├── .resources/                 # Resource Layer：设计系统资源地图
├── harmony-ui-playground/src/   # Source Layer：真实组件、Block、样式和 token
└── wiki/                       # Wiki Layer：给人看的说明文档
```

---

## 关键目录职责

| 目录 | 层级 | 作用 |
|---|---|---|
| `.agent/skills/shadcn/` | Skill Layer | 定义 AI 如何读取资源、生成页面和遵守规范 |
| `.resources/{system}/` | Resource Layer | 存放 page_type、layout、Block、Component 索引和 how to use 文档 |
| `harmony-ui-playground/src/component/` | Source Layer | Component 的真实 React 源码 |
| `harmony-ui-playground/src/blocks/` | Source Layer | 稳定 Block 模板源 |
| `harmony-ui-playground/src/styles/` | Source Layer | 样式和 token |
| `harmony-ui-playground/src/render/` | 生成预览 | AI 生成结果预览，不作为新模板源 |
| `wiki/` | Wiki Layer | 解释系统、流程、模板和验收标准 |

## 两个容易混淆的边界

### 项目 components.json 和资源层 components.json

| 文件 | 作用 |
|---|---|
| `harmony-ui-playground/components.json` | shadcn 项目配置，包含 alias、Tailwind、icon library 等工程信息 |
| `.resources/{system}/components.json` | 资源层组件索引，告诉 AI 当前设计系统有哪些 Component 可用 |

### blocks 和 render

| 目录 | 能不能作为模板源 | 说明 |
|---|---|---|
| `src/blocks/**` | 可以 | 稳定 Block 模板源 |
| `src/render/**` | 不可以 | 生成结果预览，只用于查看和验收 |
